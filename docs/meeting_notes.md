# Meeting Notes

## Objective Clarification

The user wants an AI agent that can be connected to WeChat and feel like a WeChat friend. The user wants to complain, vent, talk through pressure, and receive emotionally intelligent responses without a cold customer-service tone.

Success for the first version means:

- The user can talk to the AI from a WeChat-like entry.
- The AI feels like a believable late-night friend.
- A few invited friends can also use it safely.
- The product remembers lightweight preferences without becoming invasive.

## Target Users And Scenarios

Target users:

- Primary: the product owner.
- Secondary: a small group of invited friends.

Core scenarios:

- The user feels stressed or emo late at night and wants someone to talk to.
- The user wants to complain about work, relationships, family, friends, or daily pressure.
- The user wants the AI to understand the mood first, then optionally help untangle the situation.

## Value And Differentiation

The product should feel less like a generic chatbot and more like a personal companion with a stable personality. Its value is not only answering questions, but creating a reliable emotional rhythm: receive the vent, respond with personality, ask one useful follow-up, and help the user avoid impulsive choices.

## MVP Scope And Non-Goals

MVP scope:

- Text-based chat.
- Enterprise WeChat or WeChat customer-service style entry.
- Inner-test whitelist.
- "Late-night sharp-tongued friend" personality.
- Basic memory for each user.
- Basic emotional-risk detection and safety fallback.
- Lightweight admin console for configuration and monitoring.

Non-goals:

- Public launch for many users.
- Payment, subscriptions, or billing.
- Personal WeChat account automation.
- Medical diagnosis or therapy claims.
- Full voice, image, video, or multimodal chat.
