# Product Blueprint

## Product Definition

微信树洞 AI 是一个接入微信环境的 AI 陪聊智能体，像“嘴欠但靠谱的深夜损友”一样陪你和少数朋友吐槽、emo、睡前聊天，并通过表情包意图、短语音脚本和真人聊天节奏增强陪伴感。

## Target Users

- 你本人。
- 少数被邀请的朋友。
- 第一版不面向公开大规模用户。

## Core Scenarios

- 深夜压力：用户晚上压力很大，想找一个能接住情绪的对象说几句。
- emo 陪伴：用户不知道自己为什么烦，但希望有人能顺着聊下去。
- 吐槽复盘：用户抱怨工作、感情、人际或生活琐事，AI 先陪着吐槽，再轻轻帮用户捋清楚。
- 冲动刹车：用户情绪明显上头时，AI 从玩笑语气切换到更认真、更稳的安全模式。

## Value Proposition

这个产品的核心价值是让用户在微信环境里拥有一个稳定、熟悉、有人味的 AI 朋友。它不是普通问答机器人，也不冒充心理医生；它的重点是陪伴、吐槽、共情、复盘，以及在危险边缘及时变严肃。

为了降低用户手动提供私人聊天样本的负担，第一版会采用公开样本蒸馏路线：从许可清楚或可作为研究参考的公开项目中提炼行为规则，例如短句节奏、表情包触发、语音触发、追问方式和安全切换，而不是复制公开数据原文。

## Primary Flow

1. 用户通过企业微信或微信客服式入口向 AI 发消息。
2. 系统识别用户身份、当前情绪、话题类型和是否存在风险。
3. AI 使用“深夜损友”人格回复：先接住情绪，再用轻微毒舌和陪伴感继续对话。
4. 多模态行为层判断当前更适合纯文字、文字加表情包、短语音脚本，还是严肃安全回复。
5. 如果用户愿意继续聊，AI 用短追问推动对话，而不是一次性说教。
6. 系统在合适时记录轻量记忆，例如昵称、常见压力源、喜欢的称呼、语气边界。
7. 如果出现自伤、极端冲动、违法暴力等风险，AI 切换为严肃安全模式并提供求助建议。

## MVP Scope

- 文本聊天。
- 表情包/贴纸意图：先输出“该不该发、发哪类”，真实素材等下载和版权核准后接入。
- 短语音脚本和触发规则：先决定何时发语音、语气是什么，真实语音合成在架构阶段确认。
- 企业微信/微信客服式接入。
- 少数用户白名单。
- 用户级会话记忆。
- 人格 prompt 配置。
- 公开样本蒸馏：建立公开来源清单、样本结构和行为规则。
- 情绪和危机场景识别。
- 简单管理后台：用户列表、人格设置、记忆开关、表情包/语音策略、运行状态、最近会话摘要。
- 基础日志和错误追踪。

## Non-Goals

- 不做个人微信号自动回复。
- 不做公开注册和商业化收费。
- 不做心理诊断或治疗承诺。
- 不做视频能力。
- 不做真实人物声音克隆。
- 不直接使用未授权表情包或贴纸素材作为产品资产。
- 不下载大体积表情包、语音、视频文件，直到用户关闭 VPN 并完成来源核准。
- 不做多人格商城或内容社区。

## Open Questions

- 静态原型中的后台页面需要做到多细？
- 记忆默认保存多久？
- 内测朋友是由管理员手动添加，还是通过邀请码加入？
- 是否需要每日或睡前主动问候？
- 表情包素材最终采用自制、购买授权，还是从可商用素材库中选择？
- 语音最终采用通用合成音色，还是为产品定制一个非真人克隆的专属音色？
