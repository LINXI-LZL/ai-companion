# UI Preview

## Static Prototype

Open `web_prototype/index.html` in a browser to review the static prototype.

The prototype includes:

- WeChat-like chat experience.
- Sticker and short-voice decision preview.
- Lightweight admin console.
- Inner-test user and memory controls.
- Public sample distillation status.
- Safety mode and operational readiness page.

## Review Checklist

| Area | What To Check |
|---|---|
| Chat | The first screen should make the product feel like a real companion, not a marketing page. |
| Sticker | Sticker intent should be visible as a behavior decision, while final media files remain deferred. |
| Voice | Voice should appear as a short script and trigger rule, not as a finished voice clone. |
| Admin | Admin pages should be quiet, scan-friendly, and suitable for repeated checking. |
| Safety | High-risk content should clearly disable jokes, stickers, and playful voice. |
| Distillation | Public sample sources should show license and download status. |

## Confirmation Gate

After reviewing, confirm:

- Whether these pages are right.
- Whether the main flow feels right.
- Whether information placement is right.
- Whether architecture planning can begin.
